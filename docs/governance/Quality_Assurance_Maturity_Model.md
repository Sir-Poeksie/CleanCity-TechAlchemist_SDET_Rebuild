# QA Maturity Model – CleanCity Waste Pickup Scheduler

**Purpose:** Define QA process maturity, improvement roadmap, and capability assessment.

**Prepared by:** Mpumelelo Theophilas Nxazonke  
**Date:** 2025-11-06  
**Version:** 1.0  

---

## 1. QA Maturity Levels

| Level | Description | Indicators |
|-------|------------|------------|
| Level 1 – Initial | Ad hoc testing | Manual test cases, no RTM, limited automation |
| Level 2 – Repeatable | Basic QA processes | Test plans exist, partial automation, basic CI/CD |
| Level 3 – Defined | Standardized processes | RTM maintained, TDD/BDD applied, structured automation |
| Level 4 – Managed | Metrics-driven QA | KPIs tracked, coverage & defect trends analyzed, CI/CD integrated |
| Level 5 – Optimizing | Continuous improvement | Lessons learned, proactive risk mitigation, full automation, performance/security metrics integrated |

---

## 2. Current Status

- Level 3 – Defined: TDD + BDD implemented, RTM maintained, automation framework in place, CI/CD triggers automated tests.  

---

## 3. Roadmap to Level 5

- Increase automation coverage (unit, API, E2E, performance).  
- Integrate real-time metrics dashboards.  
- Regular security & accessibility audits.  
- Conduct quarterly process retrospectives and improvements.
